    Entity Relationship Diagram

    This page maps the current Shopwise database schema to the main domain entities used by the Django backend.

    The goal is not to document every framework table generated by Django, but to explain the business entities, the most important columns, and especially the relationship types visible in the DataGrip export.

    Version: 2.0

    Relationship notation

    • 1:1 means exactly one related row on both sides
    • 1:N means one parent row can have many child rows
    • N:1 means many rows point to one parent row
    • optional means the foreign key is nullable
    • snapshot only means the relation is stored as copied scalar fields, not as a foreign key

    Scope

    The current schema is best understood in these domains:

    • Identity and customer profile
    • Catalog and pricing
    • Cart and checkout
    • Orders, payments, and shipping
    • Promotions and audit trail
    • Supplier configuration

    Identity And Customer Profile

    User

    Registered user account based on a custom email-first Django user model.

    Field Type Notes
    id Integer Primary key
    email Email Unique login identifier
    username String Derived for Django compatibility
    email_verified Boolean Email verification state
    token_version Integer Token revocation version
    is_staff Boolean Admin access
    is_superuser Boolean Django superuser flag

    Relationships:

    • User -> CustomerProfile (1:1)
    • User -> Cart (1:N, optional on Cart)
    • User -> Order (1:N, optional on Order)
    • User -> AuditEvent (1:N, optional on AuditEvent)
    • User -> EmailVerificationToken (1:N)
    • User -> EmailChangeRequest (1:N)
    • User -> PasswordResetRequest (1:N)
    • User -> ActiveCart (1:1)

    CustomerProfile

    Extension row for e-commerce customer data.

    Field Type Notes
    id Integer Primary key
    user_id Integer Unique FK to User
    default_shipping_address_id Integer Optional FK to Address
    default_billing_address_id Integer Optional FK to Address

    Relationships:

    • CustomerProfile -> User (1:1)
    • CustomerProfile -> Address (1:N)
    • CustomerProfile -> default_shipping_address (N:1, optional)
    • CustomerProfile -> default_billing_address (N:1, optional)

    Address

    Saved customer address belonging to a profile.

    Field Type Notes
    id Integer Primary key
    profile_id Integer FK to CustomerProfile
    first_name String Recipient first name
    last_name String Recipient last name
    street_line_1 String Required
    city String Required
    postal_code String Required
    country Country code ISO country code
    phone String Required for new writes

    Relationships:

    • Address -> CustomerProfile (N:1)

    Catalog And Pricing

    Category

    Flat product taxonomy. The current schema does not contain a parent-child self reference.

    Field Type Notes
    id Integer Primary key
    name String Unique

    Relationships:

    • Category -> Product (1:N, Product.category_id is optional)
    • Category -> Discount (1:N, optional target)
    • Category -> PromotionCategory (1:N)

    TaxClass

    Named tax classification used by product pricing.

    Field Type Notes
    id Integer Primary key
    name String Display name
    code String Unique machine code
    rate Decimal Tax percentage
    is_active Boolean Active flag

    Relationships:

    • TaxClass -> Product (1:N, Product.tax_class_id is optional)

    Product

    Sellable catalog item. Pricing is in transition from legacy price to tax-aware net and gross fields.

    Field Type Notes
    id Integer Primary key
    name String Product name
    slug String Optional unique slug
    price Decimal Legacy price field
    price_net_amount Decimal Canonical net amount
    price_gross_amount Decimal Derived gross amount
    currency Enum ISO currency code
    tax_class_id Integer Optional FK to TaxClass
    stock_quantity Integer Available stock
    category_id Integer Optional FK to Category
    primary_image_id Integer Optional FK to ProductImage
    is_active Boolean Availability flag

    Relationships:

    • Product -> Category (N:1, optional)
    • Product -> TaxClass (N:1, optional)
    • Product -> ProductImage (1:N)
    • Product -> primary_image (N:1, optional)
    • Product -> CartItem (1:N)
    • Product -> OrderItem (1:N)
    • Product -> Discount (1:N)
    • Product -> PromotionProduct (1:N)
    • Product -> InventoryReservation (1:N)

    ProductImage

    Gallery image for a product.

    Field Type Notes
    id Integer Primary key
    product_id Integer FK to Product
    image File Stored asset
    alt_text String Optional alt text
    sort_order Integer Gallery ordering
    created_at DateTime Creation timestamp

    Relationships:

    • ProductImage -> Product (N:1)

    Cart And Checkout

    Cart

    Shopping cart for either an authenticated user or an anonymous visitor.

    Field Type Notes
    id Integer Primary key
    user_id Integer Optional FK to User
    anonymous_token_hash String Optional unique guest token hash
    status Enum ACTIVE, CONVERTED, MERGED
    merged_into_cart_id Integer Optional self FK
    merged_at DateTime Merge timestamp
    claimed_offer_token String Optional claimed offer token
    created_at DateTime Creation timestamp

    Relationships:

    • Cart -> User (N:1, optional)
    • Cart -> CartItem (1:N)
    • Cart -> Cart (N:1 self-reference, optional via merged_into_cart_id)
    • Cart -> ActiveCart (1:N from Cart perspective, 1:1 from User perspective through pointer table)

    CartItem

    Single product line inside a cart.

    Field Type Notes
    id Integer Primary key
    cart_id Integer FK to Cart
    product_id Integer FK to Product
    quantity Integer Positive quantity
    price_at_add_time Decimal Snapshot when item was added
    created_at DateTime Creation timestamp
    updated_at DateTime Update timestamp

    Relationships:

    • CartItem -> Cart (N:1)
    • CartItem -> Product (N:1)

    ActiveCart

    MySQL-safe pointer that guarantees a single current active cart per authenticated user.

    Field Type Notes
    id Integer Primary key
    user_id Integer Unique FK to User
    cart_id Integer FK to Cart
    created_at DateTime Creation timestamp
    updated_at DateTime Update timestamp

    Relationships:

    • ActiveCart -> User (1:1)
    • ActiveCart -> Cart (N:1)

    Orders, Payments, And Shipping

    Order

    Checkout result storing customer, address, pricing, and supplier snapshots.

    Field Type Notes
    id Integer Primary key
    user_id Integer Optional FK to User
    customer_email Email Required
    status Enum CREATED, PAID, PAYMENT_FAILED, DELIVERY_FAILED, SHIPPED, DELIVERED, CANCELLED
    shipping_* Mixed Shipping address snapshot
    billing_* Mixed Billing address snapshot
    subtotal_net Decimal Optional snapshot total
    subtotal_gross Decimal Optional snapshot total
    total_tax Decimal Optional snapshot total
    total_discount Decimal Optional snapshot total
    order_discount_gross Decimal Optional order-level discount snapshot
    order_promotion_code String Optional applied order promotion code
    currency Enum Currency of all monetary snapshots
    supplier_* Mixed Supplier identity, address, and payment snapshots
    is_claimed Boolean Claimed guest order flag
    claimed_by_user_id Integer Optional FK to User
    guest_access_token_hash String Optional guest access token
    created_at DateTime Creation timestamp

    Relationships:

    • Order -> User (N:1, optional)
    • Order -> claimed_by_user (N:1, optional)
    • Order -> OrderItem (1:N)
    • Order -> Payment (1:N)
    • Order -> Shipment (1:N)
    • Order -> InventoryReservation (1:N)
    • Order -> Supplier (no FK, snapshot only)
    • Order -> OrderPromotion (no FK, stored as order_promotion_code snapshot only)

    InventoryReservation

    Temporary stock reservation tied to an order before final stock commitment.

    Field Type Notes
    id Integer Primary key
    order_id Integer FK to Order
    product_id Integer FK to Product
    quantity Integer Reserved quantity
    status Enum ACTIVE, COMMITTED, RELEASED, EXPIRED
    expires_at DateTime Reservation expiry
    release_reason Enum Optional reason
    created_at DateTime Creation timestamp

    Relationships:

    • InventoryReservation -> Order (N:1)
    • InventoryReservation -> Product (N:1)

    OrderItem

    Immutable order line with extensive pricing snapshots captured at checkout.

    Field Type Notes
    id Integer Primary key
    order_id Integer FK to Order
    product_id Integer FK to Product
    quantity Integer Positive quantity
    price_at_order_time Decimal Legacy snapshot field
    unit_price_at_order_time Decimal Unit price snapshot
    line_total_at_order_time Decimal Line total snapshot
    unit_price_net_at_order_time Decimal Net unit price snapshot
    unit_price_gross_at_order_time Decimal Gross unit price snapshot
    tax_amount_at_order_time Decimal Tax amount snapshot
    tax_rate_at_order_time Decimal Effective tax rate snapshot
    promotion_code_at_order_time String Optional line promotion code
    promotion_discount_gross_at_order_time Decimal Optional discount snapshot
    product_name_at_order_time String Product name snapshot
    line_total_net_at_order_time Decimal Net line total snapshot
    line_total_gross_at_order_time Decimal Gross line total snapshot

    Relationships:

    • OrderItem -> Order (N:1)
    • OrderItem -> Product (N:1)

    Payment

    Single payment attempt for an order. The schema allows multiple attempts per order.

    Field Type Notes
    id Integer Primary key
    order_id Integer FK to Order
    status Enum PENDING, SUCCESS, FAILED
    payment_method Enum CARD, COD
    provider Enum DEV_FAKE, ACQUIREMOCK
    amount Decimal Optional monetary snapshot
    currency Enum Optional currency snapshot
    provider_payment_id String Optional external id
    provider_reference String Optional provider reference
    redirect_url String Optional hosted payment URL
    paid_at DateTime Optional success timestamp
    failed_at DateTime Optional failure timestamp
    failure_reason String Optional failure description
    created_at DateTime Creation timestamp

    Relationships:

    • Payment -> Order (N:1)

    Shipment

    Shipping fulfillment record attached to an order.

    Field Type Notes
    id Integer Primary key
    order_id Integer FK to Order
    provider_code String Shipping provider code
    service_code String Service code
    carrier_name_snapshot String Carrier display name at creation time
    service_name_snapshot String Service display name at creation time
    tracking_number String Optional tracking id
    status Enum PENDING, LABEL_CREATED, IN_TRANSIT, DELIVERED, FAILED_DELIVERY, CANCELLED
    label_url String Optional label URL
    label_file File Optional stored label file
    receiver_snapshot JSON Receiver snapshot data
    shipped_at DateTime Optional shipped timestamp
    delivered_at DateTime Optional delivered timestamp
    created_at DateTime Creation timestamp

    Relationships:

    • Shipment -> Order (N:1)
    • Shipment -> ShipmentEvent (1:N)

    ShipmentEvent

    Normalized tracking or carrier event belonging to a shipment.

    Field Type Notes
    id Integer Primary key
    shipment_id Integer FK to Shipment
    event_type String Provider event type
    raw_status String Optional raw carrier status
    normalized_status Enum Shopwise shipment status
    payload JSON Raw event payload
    external_event_id String Optional provider event id
    occurred_at DateTime Optional event timestamp
    processed_at DateTime Optional processing timestamp
    created_at DateTime Creation timestamp

    Relationships:

    • ShipmentEvent -> Shipment (N:1)

    Promotions And Discounts

    Discount

    Legacy discount model still present in the schema for direct product or category targeting.

    Field Type Notes
    id Integer Primary key
    name String Discount name
    discount_type Enum PERCENT or FIXED
    value Decimal Discount value
    valid_from Date Start date
    valid_to Date End date
    is_active Boolean Active flag
    product_id Integer Optional FK to Product
    category_id Integer Optional FK to Category

    Relationships:

    • Discount -> Product (N:1, optional)
    • Discount -> Category (N:1, optional)

    Promotion

    Current line-level promotion definition. Targets are assigned through explicit join tables.

    Field Type Notes
    id Integer Primary key
    name String Display name
    code String Unique business identifier
    type Enum PERCENT or FIXED
    value Decimal Discount magnitude
    amount_scope Enum NET or GROSS
    priority Integer Conflict resolution priority
    is_active Boolean Active flag
    active_from Date Optional lower bound
    active_to Date Optional upper bound

    Relationships:

    • Promotion -> PromotionProduct (1:N)
    • Promotion -> PromotionCategory (1:N)

    PromotionProduct

    Explicit join table linking a promotion to a product.

    Field Type Notes
    id Integer Primary key
    promotion_id Integer FK to Promotion
    product_id Integer FK to Product

    Relationships:

    • PromotionProduct -> Promotion (N:1)
    • PromotionProduct -> Product (N:1)

    PromotionCategory

    Explicit join table linking a promotion to a category.

    Field Type Notes
    id Integer Primary key
    promotion_id Integer FK to Promotion
    category_id Integer FK to Category

    Relationships:

    • PromotionCategory -> Promotion (N:1)
    • PromotionCategory -> Category (N:1)

    OrderPromotion

    Order-level promotion definition applied to the whole order instead of individual line items.

    Field Type Notes
    id Integer Primary key
    name String Display name
    code String Unique business identifier
    type Enum PERCENT or FIXED
    value Decimal Discount magnitude
    acquisition_mode Enum AUTO_APPLY, CAMPAIGN_APPLY, MANUAL_ENTRY
    stacking_policy Enum EXCLUSIVE, STACKABLE_WITH_LINE
    minimum_order_value Decimal Optional eligibility threshold
    priority Integer Conflict resolution priority
    is_active Boolean Active flag
    active_from Date Optional lower bound
    active_to Date Optional upper bound
    is_discoverable Boolean Messaging visibility flag

    Relationships:

    • OrderPromotion -> Offer (1:N)
    • OrderPromotion -> Order (no FK, stored as order_promotion_code snapshot only)

    Offer

    Concrete token or coupon that grants access to an order-level promotion.

    Field Type Notes
    id Integer Primary key
    token String Unique token
    promotion_id Integer FK to OrderPromotion
    status Enum CREATED, DELIVERED, CLAIMED, REDEEMED, EXPIRED
    is_active Boolean Active flag
    active_from Date Optional lower bound
    active_to Date Optional upper bound

    Relationships:

    • Offer -> OrderPromotion (N:1)

    Supplier Configuration

    Supplier

    Canonical supplier identity used for seller and invoice snapshots at order creation time.

    Field Type Notes
    id Integer Primary key
    name String Supplier name
    company_id String Optional registration number
    vat_id String Optional VAT id
    email Email Optional contact
    phone String Optional contact
    is_active Boolean Only active supplier is used for new orders
    created_at DateTime Creation timestamp

    Relationships:

    • Supplier -> SupplierAddress (1:N)
    • Supplier -> SupplierPaymentDetails (1:N)
    • Supplier -> Order (no FK, snapshot only)

    SupplierAddress

    Postal or registered address for a supplier.

    Field Type Notes
    id Integer Primary key
    supplier_id Integer FK to Supplier
    label String Optional label
    street_line_1 String Required
    city String Required
    postal_code String Required
    country String Country code or name
    is_default_for_orders Boolean Default address for order snapshots

    Relationships:

    • SupplierAddress -> Supplier (N:1)

    SupplierPaymentDetails

    Banking or payment details copied into orders as immutable snapshots.

    Field Type Notes
    id Integer Primary key
    supplier_id Integer FK to Supplier
    label String Optional label
    bank_name String Optional bank name
    account_number String Optional account number
    iban String Optional IBAN
    swift String Optional SWIFT
    is_default_for_orders Boolean Default payment details for order snapshots

    Relationships:

    • SupplierPaymentDetails -> Supplier (N:1)

    Audit Trail

    AuditEvent

    Best-effort business audit record.

    Field Type Notes
    id Integer Primary key
    entity_type String Entity name
    entity_id String Target entity id
    action String Action identifier
    actor_type Enum system, customer, admin
    actor_user_id Integer Optional FK to User
    actor_identifier String Optional external identifier
    metadata JSON Business metadata
    context JSON Additional context
    scope_key String Optional scope partition
    created_at DateTime Creation timestamp

    Relationships:

    • AuditEvent -> User (N:1, optional)

    Relationship Summary

    The key cardinalities in the current schema are:

    • User -> CustomerProfile (1:1)
    • CustomerProfile -> Address (1:N)
    • User -> Cart (1:N)
    • User -> ActiveCart (1:1)
    • Cart -> CartItem (1:N)
    • CartItem -> Product (N:1)
    • Category -> Product (1:N, optional on Product)
    • TaxClass -> Product (1:N, optional on Product)
    • Product -> ProductImage (1:N)
    • Order -> OrderItem (1:N)
    • Order -> Payment (1:N)
    • Order -> Shipment (1:N)
    • Shipment -> ShipmentEvent (1:N)
    • Order -> InventoryReservation (1:N)
    • Product -> OrderItem (1:N)
    • Product -> InventoryReservation (1:N)
    • Discount -> Product (N:1, optional)
    • Discount -> Category (N:1, optional)
    • Promotion -> PromotionProduct (1:N)
    • Promotion -> PromotionCategory (1:N)
    • PromotionProduct -> Product (N:1)
    • PromotionCategory -> Category (N:1)
    • OrderPromotion -> Offer (1:N)
    • Supplier -> SupplierAddress (1:N)
    • Supplier -> SupplierPaymentDetails (1:N)
    • AuditEvent -> User (N:1, optional)

    Snapshot-only relations that do not appear as foreign keys:

    • Order -> Supplier
    • Order -> OrderPromotion
    • Order -> customer shipping and billing address data
